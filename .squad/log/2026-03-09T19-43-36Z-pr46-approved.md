## Key Decisions

### 1. Stable ID Implementation (Coordinator)
**Decision:** Change ResistorReading.Id from Guid to deterministic string formula.

**Formula:** `det_{centerX}_{centerY}` where:
- centerX, centerY are bounding box centers quantized to 10-unit grid
- Grid: `(int)(coordinate * 100) / 10` in 0-1000 virtual coordinate space
- Example: center at (450.5, 320.8) → `det_45_32`

**Impact:** 
- Same resistor in consecutive frames receives identical ID (no flicker)
- Enables stable tracking without manual GUID management
- Applied consistently across ResistorDetectionService, ResistorOverlayDrawable, and related collections

### 2. Color Band Extraction Scope (Coordinator)
**Decision:** Exclude color band extraction from PR #46 scope.

**Rationale:**
- Color band extraction is a separate feature with its own dependencies (HSV processing)
- Service skeleton is functional without it (placeholder returns empty)
- Skipped test explicitly tracks this gap for future implementation
- PR #46 delivery unblocks downstream integration work

**Non-Blocking:**
- Documented as intentional gap, not regression
- Will be addressed in separate backlog item when HSV extraction ships

### 3. Pause/Resume Interface (Coordinator + Agent-30)
**Decision:** Add Pause() and Resume() method signatures to IResistorDetectionService.

**Impact:**
- App.xaml.cs can now pause detection during Window.Stopped
- Enables battery optimization lifecycle management (PR #53)
- Allows ResistorDetectionService to implement _isPaused flag without interface violation
